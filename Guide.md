## 웹 개발 과정 ( 프론트엔드 + 백엔드 )

# 프로젝트를 위해 필요한 선택지

```
1. 서버선택
2. 백엔드 언어 선택
3. 데이터베이스 선택
4. 아키텍트 구축
5. 프론트엔드 선택 ( SPA , 링크방식 )
```

# 기술사항

```
1. OS 선택 ( window server, linux(추천) )
	- window
	- centos ( 취업추천 )
	- ubuntu ( 추천 )개인 혼자서  ->>> 선생님

2. 웹서버 ( 웹언어를 해석할 수 있게 도와주는 역할 , 서버의 역할 )
	- apache ( 취업추천 ) 
	- nginx (떠오르는 샛별)(가볍고 빠르다)  ->>> 선생님
	- node ( 자바스크립트 ) (서버를 제어할 수 있다!! -> 서버특화는 아님..!?) socket io -> 채팅 빠르다 간편   ->>> 선생님
	
3. 백엔드
	- php ->> 선생님
	- python   ->>> 선생님
	- java & jsp ( 취업 )
	- node ->> 선생님
	- go lang (구글(알파고)) -> 지금은 수요가 없지만 앞으로 두각을 나타낼 수도...
	
4. 데이터베이스
	- oracle DB (유료, 비쌈)
	- mysql & maridb (취업) (마리아db는 mysql과 동일하면서 무료이다!) -->>선생님
	- mongodb ( NoSQL ) 빠르다 간단하다 startup??

5. 개발방식
	- restful api (취업) -->>선생님 (멀티디바이스 핸드폰, 태블릿, pc 모두 가능!)
	- MVC ( model - controller - view )
	
6. 프론트엔드
	- html, css, js  ->  필수 빠질수없어
	
7. 프론트방식
	- SPA (react(어려움), angular(백엔드 개발자가 알기 쉬움), vue(비교적 쉽다)) ( react를 배울 예정 )
	- 페이지이동 방식

8. ui / ux
	- 반응형
	- 하이브리드 ( 웹 + 모바일 웹 )
	- 네이티브 ( 웹 + 앱 )
```
